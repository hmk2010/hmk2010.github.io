golang closure(闭包)

什么是闭包？

闭包 = 匿名函数 + 外部变量引用

介绍闭包前，先来见一下匿名函数(anonymous function).

匿名函数

匿名函数：和普通函数类似，但它没有名称。它像变量一样动态创建的。

闭包函数

闭包是一个特殊的匿名函数包含外部变量的引用。

闭包的应用场景

  1. 隔离数据
    外部函数返回后，闭包函数还可以保持对数据的访问。

  2. 包装函数和创建中间件(middleware)
    函数在golang中是第一公民。不仅可以动态创建匿名函数，也可作为函数参数进行传递.

  3. sort包的二叉查找

闭包的常见陷阱和错误

  1. defer和go语句的参数是-函数调用

  2. for语句的变量通过引用传递
    for i,v := range m { v1 := v}
    //在迭代过程: i,v是相同的变量(地址一样)，只是值在变化; v1是不同的变量。

  3. go语句和闭包

理解golang的nil

golang的nil在概念上和其它语言的null、None、nil、NULL一样，都指代零值或空值。nil是预先说明的标识符，也即通常意义上的关键字。
golang中，nil只能赋值给pointer、channel、func、interface、map、slice类型的变量。如果未遵循这个规则，则会引发panic。
零值是go中变量在声明之后但是未初始化被赋予的该类型的一个默认值。

nil和nil是不能比较的

nil自身是untyped(无类型)

不同类型nil的address是一样的

不同类型的nil是不能比较的

nil是map,slice,pointer,channel,func,interface的零值

interface(接口)和nil

golang的接口类型的变量底层是作为两个成员来实现，一个是type，一个是data。type用于存储变量的动态类型，data用于存储变量的具体数据。
一个接口类型值是nil只用内部的type和data都是nil。即(nil, nil).

让我们来看一种特例：(*interface{})(nil)。将nil转成interface类型的指针，其实得到的结果仅仅是空接口类型指针并且它指向无效的地址.

error和nil

首先error是interface类型。
interface类型的变量和nil的相等比较出现最多的地方应该是error接口类型的值与nil的比较。
使用过程需要注意non-nil error的情况。

nil的有效应用

  1. 指针

  2. slice(切片)

  3. map()

  4. chan(通道)

  5. func(函数)

  6. interface(接口)

defer

defer的三个原则

  1. A deferred function’s arguments are evaluated when the defer statement is evaluated.(参数在定义时确定)
  1. Deferred function calls are executed in Last In First Out order after the surrounding function returns.(调用顺序：LIFO)
  1. Deferred functions may read and assign to the returning function’s named return values(返回前可以读取和修改函数的命名参数)

panic

panic停止程序流程并进入panicking状态。当一个函数F调用panic,F函数停止执行并正常执行所以defer函数，然后F返回到它的调用者。对于调用者，F就像调用panic一样。在当前goroutine中，函数这样一层层往上返回，执行程序崩溃的地方。

recover

recover用于恢复panicking状态的协程。只在defer函数里面使用。如果是普通执行，recover只是返回nil，对程序没有影响。
