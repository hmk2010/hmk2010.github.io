golang接口

发表于 2017-06-29

接口是什么

简单来说，接口就是一种契约，定义了一组方法(函数)的签名，描述所具有的一组行为操作（描述数据类型的行为，以及数据类型的共性特征）。

接口的定义

type Stringer interface{
String() string
}

接口的实现

在Go语言中，一个类型只需要实现了接口要求的所有函数，我们就说这个类实现了该接口。
接口和接口的实现之间不存在显示的强制声明引用。(和java，c#等语言比较)

接收者是指针(t *T)时，接口实例必须是指针
接收者是值 (t T) 时，接口实例可以是指针也可以是值

T变量包含了定义在T和T上的所有方法，T变量只包含定义在T上的方法。

所以判断一个数据类型是否实现了接口，和接收者的类型是相关的。只有实现了接口的数据类型才能赋值给接口变量和调用接口方法。

type Animal struct{
Name string
}

func (p *Animal)String() string{
//…
}

接口在面向对象思想的扮演角色

golang不支持完整的面向对象思想，它没有继承，多态则完全依赖接口实现。
golang只能模拟继承，其本质是组合，只不过golang语言为我们提供了一些语法糖使其看起来达到了继承的效果。
Go语言没有类的概念，但它支持的数据类型可以定义对应的method(s)。本质上说，所谓的method(s)其实就是函数，只不过与普通函数相比，这类函数是作用在某个数据类型上的，所以在函数签名中，会有个receiver来表明当前定义的函数会作用在该receiver上。

接口的内存结构

interface在内存上实际由两个成员组成，tab指向虚表，data则指向实际引用的数据。虚表描绘了实际的类型信息及该接口所需要的方法集。

简单理解(type ,value)

空接口

空接口是没有方法的接口。 换句话说就是所有数据类型都实现了空接口。所以空接口可以接受任何数据类型。
var i interface{}
i内部结构(nil ,nil).

接口变量只有type和value都nil时才等于nil.

类型的转化和断言

go语言是静态类型的强类型语言。不同类型之间不能直接赋值，需要手动强制类型转换，否则编译错误。

普通类型向接口类型的转换是隐式的。
接口类型向普通类型转换需要类型断言。

  1. 强制类型转换: t := T(a)

  2. Comma-ok断言的语法是：value, ok := element.(T) ;element必须是接口类型的变量，T是普通类型。如果断言失败，ok为false，否则ok为true并且value为变量的值。

  3. Comma-ok断言还支持另一种简化使用的方式：value := element.(T) ;但这种方式不建议使用，因为一旦element.(T)断言失败，则会产生运行时错误。

接口型函数，指的是用函数实现接口。我称这种函数，为接口型函数，这种方式适用于只有一个函数的接口。

golang closure(闭包)

发表于 2017-06-23

什么是闭包？

闭包 = 匿名函数 + 外部变量引用

介绍闭包前，先来见一下匿名函数(anonymous function).

匿名函数

匿名函数：和普通函数类似，但它没有名称。它像变量一样动态创建的。

闭包函数

闭包是一个特殊的匿名函数包含外部变量的引用。

闭包的应用场景

  1. 隔离数据
    外部函数返回后，闭包函数还可以保持对数据的访问。

  2. 包装函数和创建中间件(middleware)
    函数在golang中是第一公民。不仅可以动态创建匿名函数，也可作为函数参数进行传递.

  3. sort包的二叉查找

闭包的常见陷阱和错误

  1. defer和go语句的参数是-函数调用

  2. for语句的变量通过引用传递
    for i,v := range m { v1 := v}
    //在迭代过程: i,v是相同的变量(地址一样)，只是值在变化; v1是不同的变量。

理解golang的nil

发表于 2017-06-22

golang的nil在概念上和其它语言的null、None、nil、NULL一样，都指代零值或空值。nil是预先说明的标识符，也即通常意义上的关键字。
golang中，nil只能赋值给pointer、channel、func、interface、map、slice类型的变量。如果未遵循这个规则，则会引发panic。
零值是go中变量在声明之后但是未初始化被赋予的该类型的一个默认值。

nil和nil是不能比较的

nil自身是untyped(无类型)

不同类型nil的address是一样的

不同类型的nil是不能比较的

nil是map,slice,pointer,channel,func,interface的零值

interface(接口)和nil

golang的接口类型的变量底层是作为两个成员来实现，一个是type，一个是data。type用于存储变量的动态类型，data用于存储变量的具体数据。
一个接口类型值是nil只用内部的type和data都是nil。即(nil, nil).

让我们来看一种特例：(*interface{})(nil)。将nil转成interface类型的指针，其实得到的结果仅仅是空接口类型指针并且它指向无效的地址.

error和nil

首先error是interface类型。
interface类型的变量和nil的相等比较出现最多的地方应该是error接口类型的值与nil的比较。
使用过程需要注意non-nil error的情况。

nil的有效应用

  1. 指针

  2. slice(切片)

  3. map()

  4. chan(通道)

  5. func(函数)

  6. interface(接口)

defer

defer常用于简单函数来进行一些清理操作。

defer的三个原则

  1. A deferred function’s arguments are evaluated when the defer statement is evaluated.(参数在定义时确定)
  1. Deferred function calls are executed in Last In First Out order after the surrounding function returns.(调用顺序：LIFO)
  1. Deferred functions may read and assign to the returning function’s named return values(返回前可以读取和修改函数的命名参数)

panic

panic停止程序流程并进入panicking状态。当一个函数F调用panic,F函数停止执行并正常执行所以defer函数，然后F返回到它的调用者。对于调用者，F就像调用panic一样。在当前goroutine中，函数这样一层层往上返回，执行程序崩溃的地方。

recover

recover用于恢复panicking状态的协程。只在defer函数里面使用。如果是普通执行，recover只是返回nil，对程序没有影响。
